# 控制台搜索与复制：产品思考

## 用户问题

- 日志量变大后，定位错误依赖肉眼滚动，效率低
- 排查时需要把多条日志贴给模型或同事，手动整理成本高
- 控制台常驻使用场景下，键盘优先操作体验不足

## 关键设计决策

- 采用“空格分词 + AND 匹配”而不是复杂语法
  - 学习成本低，适合开发者直觉输入
- 默认支持“仅匹配项”并提供“显示全部”切换
  - 排查时先收敛，再保留上下文查看能力
- 单行复制与批量复制并存
  - 覆盖“临时分享一条”和“完整上下文多条”两种高频路径
- 引入虚拟滚动
  - 保证控制台成为长期会话面板时仍可流畅使用

## 为什么这样做

- 关键词高亮 + 自动滚动首个命中，能把“输入→定位”路径压缩到一次操作
- 复制文本保持 `时间 + 级别 + 来源`，减少二次描述和上下文丢失
- `Ctrl/Cmd + K / C` 与 `Esc` 组合，符合多数开发工具肌肉记忆

## 已知局限

- 当前匹配权重未排序，主要按时间流顺序展示
- 高亮仅作用于 message 文本，不含 source 字段
- 复制兼容性依赖浏览器权限策略，受用户环境影响

## 后续方向

- 支持 `key:value` 检索（如 `level:error source:runtime`）
- 支持保存常用过滤器（Saved Views）
- 支持导出选中日志为结构化 JSON/Markdown
