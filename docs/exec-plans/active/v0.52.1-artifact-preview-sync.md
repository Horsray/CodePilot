# Artifact Preview (v0.52.1) 吸收合并计划

## 1. 目标
以当前分支 (`v0.51.1` fork) 为基线，吸收官方 `v0.52.1` 中引入的 Artifact 可视化预览能力，同时**坚决保留**我们独有的 `Diff` 代码审查流（AgentTimeline、FileReviewBar）。

## 2. 核心原则
- **拒绝 UI 覆盖**：绝不使用官方的 Artifact 卡片替换我们在聊天流中现有的 `Modified X files` 明细和全局 `FileReviewBar`。
- **剥离预览引擎**：仅提取官方的 `PreviewPanel` 及其底层依赖（`JSXPreview`、`WebPreview`、`CodeBlock`、Sandpack、Markdown Editor）。
- **无缝嫁接**：在我们的 `FileReviewBar`（底栏审查条）和 `AgentTimeline`（步骤明细）中，为受支持的文件类型增加【🚀 预览】入口，点击后在右侧拉起官方的 `PreviewPanel`。

## 3. 执行步骤

### Phase 1: 环境准备与基线同步
1. 运行 `git fetch upstream` 获取官方最新代码。
2. 基于当前分支创建同步分支：`git checkout -b sync/v0.52.1-artifact-preview`
3. 运行 `npm run sync:report:write` 生成冲突报告，明确改动边界。

### Phase 2: 依赖与基建合并 (无冲突部分)
1. **合并 package.json**：吸收官方新增的依赖（如 `@streamdown/*`, `react-jsx-parser`, `sandpack` 等），并执行 `npm install`。
2. **引入 AI Elements**：将官方新增的 `src/components/ai-elements/` 下的新预览组件（`jsx-preview.tsx`, `web-preview.tsx`, `sandbox.tsx` 等）原样拷入。
3. **合并 API 路由**：吸收官方用于支持预览的后端 API（如 `/api/files/preview/route.ts`, `/api/files/serve/route.ts` 等）。

### Phase 3: 核心面板合并 (精细化冲突解决)
1. **PreviewPanel 移植**：合并 `src/components/layout/panels/PreviewPanel.tsx`，保留官方的渲染逻辑。
2. **PanelZone 注册**：在 `src/components/layout/PanelZone.tsx` 和全局状态 `usePanelStore` 中，注册并确保 `PreviewPanel` 能够被正确唤起。
3. **FileTree 增强**：合并 `src/components/project/FileTree.tsx`，吸收官方底部的“新建文件/文件夹”按钮，同时**保留**我们现有的右键菜单能力。

### Phase 4: 交互嫁接 (我们的独有改造)
1. **改造 FileReviewBar**：
   - 修改 `src/components/chat/FileReviewBar.tsx`。
   - 在每行文件的操作区（现有【看 Diff 眼睛图标】旁边），判断如果该文件是可渲染类型（`.tsx`, `.html`, `.md`, `.csv` 等），则额外渲染一个【🚀 预览 (Play)】图标。
   - 点击该图标，调用 `openPreviewTab(filePath)` 拉起右侧的 `PreviewPanel`。
2. **改造 AgentTimeline**：
   - 在单步的文件修改记录（`DiffPreview`）中，同样追加【预览】按钮逻辑。

### Phase 5: 验证与验收
1. 运行 `npm run test` 确保类型与基础逻辑无回归。
2. 启动服务，让 AI 生成一个简单的 React 组件。
3. **验收标准**：
   - 聊天流中依然展示我们的 `AgentTimeline` 增删明细。
   - 底部弹出 `FileReviewBar`，点击【眼睛】能看 Diff。
   - 点击【🚀 预览】，右侧成功渲染出真实的 React UI。
   - 手机端桥接、定时任务等核心后台功能未受影响。

## 4. 风险预案
- 如果官方的 `JSXPreview` 存在沙盒安全问题或样式污染，降级为仅支持 HTML/Markdown 预览。
- 如果文件树的底部按钮与我们的滚动区域冲突，优先保留右键菜单，微调底部按钮的 Flex 布局。
