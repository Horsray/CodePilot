# Trae 风格 AI 交互重构方案 (针对 HueyingAi 宽屏对话流)

本方案旨在解决用户项目中“思考过程”与“工具调用”堆叠感强、节奏混乱的问题，通过引入 Trae 的**原子化组件**与**线性时序节奏**，重构对话流。

## 1. 核心交互节奏 (The Rhythm)

Trae 的节奏感源于“**因果链条**”的视觉显化。

| 阶段 | 动作 (Action) | 视觉反馈 (Visual) | 节奏描述 |
| :--- | :--- | :--- | :--- |
| **触发** | 用户发送指令 | 输入框收起，消息流顶部出现占位。 | 立即响应，无延迟感。 |
| **推理** | `ThoughtBlock` 流式输出 | 采用等宽字体，背景微弱闪烁，模拟大脑运作。 | 建立信任，让用户知道 AI 在“想”什么。 |
| **行动** | `ToolCard` 弹出 | 思考块收缩，工具卡片以动画形式滑入，显示图标与操作目标。 | 从虚（思考）到实（行动）的转换。 |
| **结果** | `Response` 最终文本 | 文本以 Markdown 形式流式呈现，引用工具执行的结果。 | 总结陈词，完成闭环。 |

## 2. 组件重构 (Component Refactor)

### 2.1 思考块 (ThoughtBlock) - 替代现有文本段落
*   **外观**：左侧带有垂直蓝色装饰条，背景色为 `#1e1e1e`（比主背景深 2%）。
*   **节奏**：AI 开始思考时自动展开；工具调用开始后，思考块高度收缩至 40px，仅保留“Thought for 5s”标签。
*   **代码参考**：
    ```tsx
    <ThoughtBlock status="streaming | completed" duration="5s">
      {/* 这里的文本应该是 Chain of Thought，而不是回复正文 */}
      "我需要先分析当前项目的目录结构，定位到 ChatView 组件..."
    </ThoughtBlock>
    ```

### 2.2 工具卡片 (ToolCard) - 替代现有绿色圆点列表
*   **外观**：放弃简单的图标+文字。采用圆角矩形卡片，包含：
    *   **左侧**：工具类型图标（如：🔍 搜索, 📄 读取, 🐚 终端）。
    *   **中间**：操作描述（如：`read_file: page.tsx`）及参数摘要。
    *   **右侧**：状态指示（转圈 -> 绿勾/红叉）及耗时。
*   **节奏**：多个工具连续调用时，采用**垂直堆叠且可折叠**的 Group 模式。

### 2.3 文件 Diff 查看 (DiffView) - 深度融合
*   **位置**：在对话流中直接嵌入内联 Diff，或者触发主工作区的 Diff 模式。
*   **交互**：提供“Accept”和“Reject”浮动按钮，且按钮带有光晕动效。

## 3. 视觉对比 (Before vs. After)

| 特性 | 当前项目 (HueyingAi) | Trae 风格重构 |
| :--- | :--- | :--- |
| **思考展示** | 混在正文中的普通段落 | 独立的、可折叠的等宽字体块 |
| **工具调用** | 简单的绿色圆点列表 | 带有图标、参数、状态的原子化卡片 |
| **信息密度** | 垂直空间占用大，信息杂乱 | 线性、模块化，重要信息高亮 |
| **节奏感** | 所有信息一股脑涌出 | 思考 -> 动作 -> 结果的严格时序 |
